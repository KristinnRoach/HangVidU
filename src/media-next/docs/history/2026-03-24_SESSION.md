# Session 2026-03-24

## Goal

Implement the first real end-to-end lab slice for `media-next` with testable playback and capture primitives.

## What changed

- defined and committed source schemas
- defined and committed runtime state
- added session workflow rules, including commit checkpoints
- implemented real remote URL playback through a focused playback lab page
- implemented real camera, microphone, and screen capture through a focused capture lab page
- split the old mixed `media-lab` into:
  - hub page
  - playback page
  - capture page
- added `mediaType`, optional `codecHints`, and playback `loading`
- replaced the camera-specific capture runtime with a generic capture runtime

## Verified

- `pnpm build` passes
- remote playback was verified manually in-browser
- capture failure states are surfaced explicitly
- manual verification steps were documented for camera, microphone, and screen-sharing flows

## Key decisions

- keep `PlayableSource` and `StreamSource` separate
- keep playback and live-stream separate domains
- keep the DOM/media-device layer in thin runtimes outside the controllers
- treat `codex/media-module-sketch` as the current integration branch
- prefer focused lab pages over one mixed experimental page

## Remaining open questions

- whether to formalize the current runtimes in `ADAPTERS.md`
- how to model the composition/session layer for common call scenarios
- when to add audio-only playback to the focused playback lab
- whether system-audio capture should become its own focused primitive/runtime

## Next sensible step

Document the adapter layer explicitly and decide whether `ADAPTERS.md` should be created before expanding more runtime behavior.
